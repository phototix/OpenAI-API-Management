OpenAI: uses /v1/organization/costs with start_time/end_time. Anthropic: uses GET https://api.anthropic.com/v1/organizations/usage_report/messages with starting_at/ending_at. Deepseek: uses GET https://api.deepseek.com/user/balance. Grok (xAI): uses GET https://management-api.x.ai/v1/billing/teams/{team_id}/prepaid/balance. Google AI Studio (GDC Billing): consults Cloud Billing Pricing API for SKU pricing. Note: Pricing API does not expose per-account credit balances.

OpenAI API Notes

  • Ensure your API key has access to the /v1/organization/costs endpoint.
  • Usage data is fetched based on the selected date range (1 day, 3 days, 7 days, or 1 month).
  • OpenAI may have rate limits; avoid excessive refreshes to prevent temporary blocks.
  • Reference: OpenAI API Documentation for Organization Costs endpoint.

DeepSeek AI API Notes

  • Ensure your API key has access to the GET /user/balance endpoint.
  • DeepSeek provides a straightforward balance retrieval without date range parameters.
  • Reference: DeepSeek API Documentation for User Balance endpoint.

xAI Grok API Notes

  • Ensure your API key has access to the /v1/billing/teams/{team_id}/prepaid/balance endpoint.
  • Replace {team_id} in the URL with your actual Team ID associated with your Grok account.
  • Reference: xAI Grok Management API Documentation for Billing and Balance endpoint.

Google AI Studio via GDC Billing

  • Enable the Cloud Billing Catalog API (Pricing API) in your Google Cloud project.
  • Create an API key or an OAuth token with access to the Pricing API. Restrict it to your project.
  • Find your Billing Account ID (format like 0123AB-456789-ABCDEF) in Google Cloud Console > Billing.
  • Limitations: The Pricing API provides SKU pricing, not your remaining credits or balance. For spend tracking, use Cloud Billing export to BigQuery or Budgets.
  • Reference: Google Cloud Billing Pricing API REST (Services and SKUs list endpoints).

Claude/Anthropic AI API Notes

  • Ensure your API key has access to the GET /v1/organizations/usage_report/messages endpoint.
  • Usage data is fetched based on the selected date range using starting_at and ending_at parameters.
  • Reference: Anthropic API Documentation for Organizations Usage Report endpoint.
